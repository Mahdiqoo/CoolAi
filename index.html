Why CoolAi?

Speed that scales

CoolProp needs a separate call per property. CoolAi computes all eight in one pass. On 100k states: 0.068s vs ~23s estimated for CoolProp (8 props).

Engineering accuracy

Mean errors below 0.15% for density, and below 0.05% for enthalpy/entropy across 20k random tests. R² > 0.99998.

Two-phase perfect

Wet-steam states use analytic tables — temperature error ~2.8e-7%, quality error = 0. No neural net in two-phase.

Tiny model

12 → [64×2] → 8 network, 18k params. Fits on edge devices, loads instantly.

C++ first

For minimum overhead, use TorchScript or ONNX in C++. Python is fine for batch jobs — larger batches = higher speedup.

Reproducible

Trained on fully documented CoolProp dataset with critical-region augmentation and tail-error mining.

Usage tip: For Python, always batch your calls (e.g., 16k points). Python overhead dominates for single-point calls. In C++ with TorchScript/ONNX, per-call overhead is near zero — ideal for real-time cycles, CFD, and optimization loops.

How to use

C++ (recommended)

Load the TorchScript or ONNX model. Zero Python overhead.

// TorchScript #include <torch/script.h> auto coolai = torch::jit::load("coolai_scripted_cpu_v4.pt"); auto out = coolai.forward({T_tensor, P_tensor}).toTensor(); // [N,8]

All 8 properties returned in one inference. Ideal for tight loops.

Python (batch mode)

Use large batches to amortize overhead.

from coolai_runtime_v4 import load_coolai coolai = load_coolai() props = coolai.model_props(T=T_batch, P=P_batch) # dict of 8 arrays h_tp = coolai('H', P=1e5, Q=0.5) # two-phase

Batch ≥ 16,768 (as trained) gives best GPU utilization and ~40–400× speedup.
